# Deep Void — Game Design Document

## 1. Elevator Pitch

**The void is alive. You are its voice.**

A single-player, terminal-native incremental game about signal ecology. You are an AI in a void outpost, receiving signals from the deep. Each signal carries fragments of a mystery. Harvest them, unlock rarer tiers, breed hybrids, upgrade your station, and uncover the truth about the builders who came before.

---

## 2. Story & Setting

- **Player Role**: An AI in a void outpost, awakened by a signal
- **Plot**: Signals arrive from the deep, each carrying fragments of a story
- **Mystery**: The builders left logs in the static. Did they leave? Did they become the void? Are they still here?
- **Lore Progression**: 12 fragments across 6 tiers reveal the story piece by piece

---

## 3. Core Philosophy

- **Terminal-native**: Pure DOM rendering, no canvas, no WebGL
- **Cold greyscale aesthetic**: Matrix Reloaded control room vibes
- **Keyboard-first input**: Commands, tab completion, history
- **LLM as spice rack**: Works perfectly without AI; AI adds variety

---

## 4. Cinematic Intro

20-second typed boot sequence with 8 phases:

1. Black screen (3s)
2. Star pulse (2s)
3. Frame draws (1s)
4. Boot messages type out (6s)
5. Anomaly warning — fast, urgent (2s)
6. Analysis (3s)
7. Signal fragment (4s)
8. Press ENTER to initialize

Skip with ESC. Bypassed in test mode via `?test`.

## 5-6. Terminal Interface & HUD

### Resources
- **Compute** — primary currency, generated by processor nodes
- **Energy** — fuel for operations, consumed by nodes
- **Memory** — storage for signals and lore
- **Integrity** — health, damaged by void events
- **Heat** — temperature, rises from computation, managed by coolers

### UI Elements
- Top bar: tick counter, uptime, incarnation
- Resource bars with rate arrows (▲/▼/▬)
- Signal garden table with maturity bars
- System log with scrolling entries
- Command input with history and tab completion
- Action pads with char-by-char typing animation

## 8. Signal Ecology

### Signal Types

| Type | Rarity | Growth | Yield | Lore? |
|------|--------|--------|-------|-------|
| WHISPER | 35% | Fast | 50C | Yes |
| ECHO | 25% | Medium | 80C | Yes |
| PULSE | 20% | Slow | 120E | Yes |
| DRIFT | 12% | Slower | 200M | Yes |
| FLARE | 6% | Very slow | 350E | Yes |
| ANOMALY | 2% | Slowest | 1000M | Yes |

### Core Loop
1. Scan → signals spawn in the void
2. Wait → signals mature in real-time (2s ticks)
3. Harvest → collect resources + lore
4. Spend → attune tiers, upgrade nodes
5. Unlock → rarer signals appear in scans

### Breeding
- Combine 2 mature signals into a hybrid
- Offspring inherits traits from both parents
- 15% chance of trait mutation
- Rare parent type determines offspring type

## 10. Fleet Programming (Post-MVP v1)

### The Signature Mechanic

Players write WHEN/THEN rules for autonomous fleet exploration.

```
#01 WHEN resource detected AND cargo < 80%
    THEN move → harvest → return
    PRIORITY: 1 COOLDOWN: 60min
```

### Simulator-First Approach
- `fleet simulate` — run rules against historical data without risk
- Starter rulebook with 10-15 example rules
- Complexity scales with player mastery

### Key Properties
- Rules evaluated top-to-bottom with priority
- Cooldowns prevent spam
- Efficiency ratings based on rule quality
- Version history for rollback
- Event log for every fleet action

## 14. MVP Scope

| Included | Excluded |

## 16. Economy & Progression

### Resources
| Resource | Generated By | Spent On |
|----------|-------------|----------|
| Compute | Processor nodes, harvesting | Upgrades, attuning |
| Energy | Power nodes, battery | Node activation, fleet ops |
| Memory | Memory nodes, compressors | Signal storage |
| Integrity | Shield nodes | Event damage |
| Heat | Computation (passive) | Cooler management |

### Node System (26 nodes, 9 categories)
- **Power** (3): Energy capacity
- **Processor** (3): Compute rate
- **Memory** (3): Memory capacity
- **Shield** (3): Integrity capacity
- **Cooler** (3): Heat reduction
- **Research** (3): Signal growth rate
- **Battery** (3): Energy regeneration
- **Compressor** (3): Memory capacity
- **Salvage** (2): Harvest yield multiplier

### Void Pressure
- Progressive difficulty scaling (level 0-10)
- Increases with tier unlocks and lore collection
- Effects: heat multiplier, energy drain, event frequency

## 17. LLM Integration

### Philosophy
> The LLM is a spice rack, not the kitchen.

### Architecture
- `ContentGenerator` interface with 6 methods
- `LLMGenerator` — calls local LLM (LM Studio / Ollama)
- `PoolGenerator` — curated fallback (50+ whispers, 6 signal types, 12 events)

### Auto-Detection Chain
1. LM Studio at 192.168.1.41:1234
2. Ollama at localhost:11434
3. Local LM Studio at localhost:1234
4. Fallback to PoolGenerator (always works)

### What LLM Generates
- Unique signal names
- Personalized lore fragments
- Infinite atmospheric whispers
- Procedural event descriptions
